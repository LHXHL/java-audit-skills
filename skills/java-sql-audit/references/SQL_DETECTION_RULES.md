# SQL 判定规则

本文件定义抽象判定，不列具体框架或库 API。

## sink 成立条件

必须同时存在：

1. 真实 `SQL_EXECUTOR`。
2. `REQUEST_PARAM` 到达执行点。
3. 参数影响查询值、标识符、排序、过滤、函数、片段或完整查询。
4. 防护缺失、不足或仍需确认。

## 参数位置

| 位置 | 风险关注 |
|------|----------|
| 值位置 | 是否存在真实 `VALUE_BINDING` 或强类型转换 |
| 标识符 | 是否由闭合集合映射产生 |
| 排序字段 | 是否由闭合集合映射产生 |
| 排序方向 | 是否限制在固定枚举 |
| 查询片段 | 是否允许外部输入控制语法结构 |
| 完整查询 | 是否允许外部输入控制查询文本 |

## 结论状态

| 状态 | 条件 |
|------|------|
| 确认漏洞 | 可控输入到达 SQL 结构或未绑定值位置，防护缺失，入口可达 |
| 条件成立 | 需要特定配置、分支、角色、环境或数据状态 |
| 待验证 | 关键运行条件或输入来源未完全确认 |
| 不可确认 | 缺源码、缺实现、缺配置或 helper 证据不足 |
| 非漏洞 | 证据显示不可控、不可达或防护充分 |

## 降级条件

- 缺入口或端到端调用链。
- sink 只来自命名推测。
- 参数被服务端常量、枚举或严格映射覆盖。
- 动态片段无法确认真实执行。
- 只有候选文件或字符串线索。

## 输出边界

报告只写项目真实证据和结论，不写通用规则名、内部测试过程或组件版本建议。
