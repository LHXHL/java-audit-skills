# SQL 注入审计报告模板

生成 `java-sql-audit` 报告前必须读取本文件。模板用于真实代码审计输出，不得诱导编造漏洞、PoC、CVSS、CVE 或修复版本。

## 硬约束

1. 报告必须且只能包含 6 个编号章节和末尾 `## 输出自检`。
2. 所有 `【填写】` 占位符必须替换；无内容时写 `无`、`N/A` 或“未确认”。
3. 不得输出 CVE、CVSS、修复版本或声称验证成功的结果。
4. 不得输出破坏性 payload、堆叠查询、命令执行、时间延迟或数据外带请求。
5. 没有确认入口时，不得给可复制攻击请求；只能写授权验证思路。
6. 候选风险必须标注状态，不能把“待验证/不可确认”写成“确认漏洞”。
7. 同根因多入口遵守 `../java-shared/VULNERABILITY_GROUPING.md`。
8. 编号章节标题必须与模板完全一致：`## 1. 审计概述`、`## 2. 结论统计`、`## 3. SQL 操作映射`、`## 4. 候选风险与非漏洞依据`、`## 5. 风险详情`、`## 6. 审计结论`。
9. 不得新增 `## 0`、`## 7`、`## 调用链表`、`## 限制与证据缺口` 等额外编号章节；额外信息必须放入模板已有章节。
10. 结论状态必须使用中文枚举：确认漏洞、条件成立、待验证、不可确认、非漏洞；不得使用 `confirmed_vulnerable`、`not_vulnerable`、`unconfirmed`。
11. 报告全文禁止出现三个连续英文句点，包括代码块、SQL 片段、路径、表格和自检。需要省略非关键字段时写中文“省略非关键字段”，或只保留已确认的最小片段。

## 文件命名

```text
{project_name}_sql_audit_{YYYYMMDD_HHMMSS}.md
```

- [ ] 报告全文没有三个连续英文句点。
- [ ] 同根因多入口已聚合，不同条件已拆分。
- [ ] 限制说明清楚标注缺失源码、缺失实现、未确认数据库或未确认入口。
