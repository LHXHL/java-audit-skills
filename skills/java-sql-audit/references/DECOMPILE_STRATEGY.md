# SQL 专项反编译策略

只在源码缺失、不完整、上游 trace 指向不可读实现，或结构化候选来自编译产物时读取本文件。

## 反编译目标

优先补齐：

- `ENTRY_ROUTE` 到 `SQL_EXECUTOR` 的直接调用链节点。
- 查询构造器、公共数据访问封装、动态排序/过滤封装。
- 参数对象、字段映射、配置映射和接口实现。
- `QUERY_TEMPLATE`、`QUERY_FRAGMENT`、`DYNAMIC_ORDER_FIELD` 的真实消费方。

不要反编译整个项目，除非无法定位候选边界。

## 证据要求

反编译结果必须记录：

- 来源产物路径。
- 生成的可读源码路径。
- 是否保留行号或只能定位方法级。
- 仍缺失的实现、配置或参数名。

只有字符串线索或类名线索时，只能作为候选，不能作为确认漏洞证据。

## blocked

无法取得关键实现时，在报告和 coverage 中写：

- `item_id`
- `item_type`
- `reason`
- `evidence`
- `next_action`
