# 路由调用链追踪报告模板（完整版）

适用场景：命中 sink、分支复杂、用户点名、pipeline 下游会读取，或需要完整证据的入口。

硬约束：

1. 输出必须包含 8 个主章节：`0` 到 `7`。
2. 所有占位符必须替换；无内容写 `无` 或 `未提供`。
3. 不得写漏洞已确认、攻击成功、风险评分、修复版本或专项验证结论。
4. 请求示例只能说明参数结构，不能生成攻击字符串、复现步骤或专项验证材料。
5. 批量、高风险或重复模式必须引用 trace helper 证据。
6. 不得凭缺失实现、业务命名或内部方法名推断具体 sink；缺实现写 `UNCONFIRMED`。
7. 数量、行号和覆盖率必须精确；无法确认写 `不可确认` 或 blocked。
