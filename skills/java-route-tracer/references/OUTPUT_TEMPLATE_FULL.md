# 路由调用链追踪报告模板（完整版）

适用场景：命中敏感 sink、分支复杂、用户点名、pipeline 下游会读取，或需要完整证据的入口方法。

硬约束：

1. 输出必须包含 8 个主章节：`0` 到 `7`。
2. 章节顺序不得调整。
3. 所有 `【填写】` 必须替换为真实内容；无内容写 `无` 或 `未提供`。
4. 不得写漏洞已确认、攻击成功、风险评分、修复版本或专项验证结论。
5. 请求示例只能说明参数结构，不能生成攻击字符串、复现步骤或专项验证材料。
6. 文件建议放在 `route_tracer/{route_slug}/`。
7. 请求模板中禁止使用 `${param}`；结构占位只允许 `{{param}}`，安全样例请求必须填入实际低风险值。
8. 不得凭 Manager/DAO/Repository 方法名推断 SQL/FILE/XML 等 sink；缺实现时写 `UNCONFIRMED`。
9. 报告只能包含模板定义的 8 个主章节，不得添加 `## 输出自检`、模型验收、技能源校验或测试过程说明。
10. 文件中禁止使用三个连续英文点号作为省略；代码片段必须摘录真实语句，不得用省略号占位。
11. 数量、行号、接口数、方法数和覆盖率必须是精确可复核值；不得写 `约`、`~`、`大约`、`60+`、尾随加号或范围数量。无法精确统计时写“未精确统计”或“不可确认”。
12. 不得新增 `可利用性`、`风险评价`、`建议立即修复`、漏洞编号或专项验证结论；只能在第 7 节写“建议下游 skill”。
13. 不得写内部规则名、`skill 规则`、规则编号、`同上` 或测试过程；不得把非调用链认证/配置基线写进“仍需确认”或用来说明“不属于本范围”。
