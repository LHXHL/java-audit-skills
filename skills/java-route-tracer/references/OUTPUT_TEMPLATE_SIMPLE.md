# 路由调用链追踪报告模板（简化版）

适用场景：未命中敏感 sink、链路很短、重复模式明显，或多方法入口中非重点方法。简化版仍必须独立追踪当前方法，不能复用其他方法结论。

硬约束：

1. 输出必须包含 5 个主章节：`0` 到 `4`。
2. 所有 `【填写】` 必须替换为真实内容。
3. 不能把 sink 可达写成漏洞成立。
4. 文件建议放在 `route_tracer/{route_slug}/`。
5. 请求模板禁止使用 `${param}`；结构占位只允许 `{{param}}`，安全样例必须用实际低风险值。
6. Manager/DAO/Repository 方法名不是 sink 证据；实现缺失时写 `UNCONFIRMED`。
7. 报告文件必须包含文件尾部的 `## 输出自检`；终端最终响应不能替代文件内自检。
8. 文件中禁止使用三个连续英文点号作为省略；代码片段必须摘录真实语句，不得用省略号占位。
