# 单入口轻量追踪模板

适用场景：用户只要求一个入口、调用层级少、无批量候选。若发现复杂分发或重复模式，应升级为 helper 追踪。

硬约束：

1. 不得写漏洞结论。
2. 不得用业务命名推断 sink。
3. 人工追踪必须写明为何不需要 helper。
4. 有结构化输出目录时仍要写 `trace_sinks.jsonl` 或 blocked。
