# 多方法追踪索引模板

适用场景：一个路由、WebService endpoint、Servlet dispatcher、Struts 通配符或批次项对应多个真实入口方法。

硬约束：

1. 输出必须包含 5 个主章节：`0` 到 `4`。
2. 方法清单必须列出全部真实入口方法，不得省略。
3. 链接必须指向真实存在的方法报告。
4. 索引不写漏洞结论，只汇总 sink 和可控性证据。
5. 方法报告若只追到缺源码的 Manager/DAO 调用，索引中的 Sink 摘要必须写 `UNCONFIRMED` 或“待确认”，不得写推断 SQL/Hibernate。
6. 索引文件必须包含文件尾部的 `## 输出自检`；终端最终响应不能替代文件内自检。
7. 文件中禁止使用三个连续英文点号作为省略。

## 输出自检

- [ ] 5 个主章节均存在且顺序正确。
- [ ] 方法清单无省略，数量与覆盖统计一致。
- [ ] 每个链接指向真实报告文件。
- [ ] 不含未替换的填写占位、dollar-brace 变量占位或英文省略号占位。
- [ ] Sink 摘要没有把缺源码的 Manager/DAO 调用写成已确认 SQL/FILE/XML sink。
- [ ] 未写漏洞已确认、CVSS、CVE 修复版本或破坏性 PoC。
