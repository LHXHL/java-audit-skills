# XXE 审计报告模板

生成 `java-xxe-audit` 报告前必须读取本文件。模板用于真实代码审计输出，不得诱导编造漏洞、PoC、CVSS、CVE、修复版本或外带验证结果。

## 硬约束

1. 报告必须且只能包含 6 个编号章节和末尾 `## 输出自检`。
2. 编号章节标题必须完全一致：`## 1. 审计概述`、`## 2. 结论统计`、`## 3. XML 解析器映射`、`## 4. 候选风险与非漏洞依据`、`## 5. 风险详情`、`## 6. 审计结论`。
3. 不得新增 `## 0`、`## 7`、`## 调用链表`、`## 限制与证据缺口` 等额外编号章节。
4. 所有 `【填写】` 占位符必须替换；无内容时写 `无`、`N/A` 或“未确认”。
5. 结论状态必须使用中文枚举：确认漏洞、条件成立、待验证、不可确认、非漏洞。
6. 不得输出 CVE、CVSS、修复版本、真实文件内容、OOB 服务器、破坏性 payload、可直接攻击请求或声称验证成功的结果。
7. 没有真实入口或授权环境时，不得给可复制攻击请求；只能写授权验证建议。
8. 授权验证建议不得指导构造 DOCTYPE、实体声明、外部实体引用、OOB/回连目标、文件读取目标或可复制请求；只写补证步骤和观测条件。
9. 报告全文禁止出现三个连续英文句点。
10. WebService/SOAP 入口地址必须引用 CXF/JAX-WS 配置、WSDL、注解服务名或 route-mapper 证据；只知道类名或方法名时，路由/入口写“未确认”，不得拼出伪 path。
11. `XStream.fromXML`、项目封装的 `XmlUtil.fromXML` 仍是 XML 解析候选；未确认底层 driver、版本或安全配置时不得直接写“非漏洞”。
12. 同根因多入口遵守 `../java-shared/VULNERABILITY_GROUPING.md`。

## 文件命名

```text
{project_name}_xxe_audit_{YYYYMMDD_HHMMSS}.md
```
