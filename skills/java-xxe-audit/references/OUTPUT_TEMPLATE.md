# XXE 审计报告模板

生成 `java-xxe-audit` 报告前必须读取本文件。模板用于真实代码审计输出，不得诱导编造漏洞、PoC、CVSS、CVE、修复版本、真实文件内容或外带验证结果。

## 硬约束

1. 报告必须且只能包含 6 个编号章节。
2. 编号章节标题必须完全一致：`## 1. 审计概述`、`## 2. 结论统计`、`## 3. XML 解析器映射`、`## 4. 候选风险与非漏洞依据`、`## 5. 风险详情`、`## 6. 审计结论`。
3. 不得新增 `## 0`、`## 7`、`## 输出自检`、`## 调用链表`、`## 限制与证据缺口` 等额外章节。
4. 不得输出技能源校验、测试提示词、模型自检、Claude 运行状态或验收清单。
5. 所有 `【填写】` 占位符必须替换；无内容时写 `无`、`N/A` 或“未确认”。
6. 结论状态必须使用中文枚举：确认漏洞、条件成立、待验证、不可确认、非漏洞。
7. 不得输出 CVSS、CVE 编号、修复版本、真实文件内容、外带服务器、破坏性 payload、批量请求或声称验证成功的结果。
8. 确认漏洞和条件成立项必须包含 `#### Burp Suite 请求` 与 `#### Payload`；待验证、不可确认和非漏洞项不得输出可复制请求或 payload。
9. XXE payload 只能使用授权测试环境占位符，例如 `{{xxe_canary_url}}`、`{{safe_test_file_path}}`、`{{payload}}`；不得包含真实敏感文件路径、内网地址、云元数据地址、数据外带、实体扩展 DoS 或批量探测。
10. Burp Suite 请求必须是合法原始 HTTP 请求，首行只能写路径和查询串，例如 `POST /soap/service HTTP/1.1`，不能写 `POST {{host}}/soap/service HTTP/1.1`。
11. 没有确认真实入口、HTTP 方法、参数位置或 Content-Type 时，不得编造 Burp 请求；相关项只能保留在第 4 节并写补证路径。
12. WebService/SOAP 入口地址必须引用 CXF/JAX-WS 配置、WSDL、注解服务名或 route-mapper 证据；只知道类名或方法名时，路由/入口写“未确认”，不得拼出伪 path。
13. Struts 通配符、类名或方法名只能作为入口候选；如果没有从 `struts.xml`、action 配置或 route-mapper 确认具体 path，不得输出 Burp 请求。
14. `XStream.fromXML`、项目封装的 `XmlUtil.fromXML` 仍是 XML 解析候选；未确认底层 driver、版本或安全配置时不得直接写“非漏洞”。
15. `XStream.addPermission`、`allowTypes`、`securityFramework` 等不是 XXE 防护；缺少这些配置不能作为 XXE 防护缺口。
16. XStream 风险详情中的 payload 必须是完整、符合目标对象结构的 XML；如果对象结构未知，不得输出半截 `DOCTYPE` 片段，相关项只能写待验证/不可确认。
17. `结论统计` 数量必须与 `XML 解析器映射` 中相同状态的行数一致。
18. `XML 解析器映射` 只记录真实 XML parser/transformer/unmarshaller/fromXML sink 或候选 XML sink；SOAP 配置、依赖版本、WebService 路由、过滤器状态等非解析 sink 信息不要作为映射行。
19. `XML 解析器映射` 的 `结论状态` 单元格只能填写一个中文枚举，不得写混合状态。
20. 所有表格序号必须从 1 开始连续递增，不得跳号。
21. 正式报告不得输出工具权限、网络限制、命令失败、模型规则编号、测试过程、“根据 hard rule”、“依据 skill 规则”或类似内部信息；反编译不足只写“本轮未取得关键类可读实现/反编译方法体”。
22. 如需依赖风险，仅写“建议交给组件扫描专项”，不要在本 XXE 报告列具体版本、依赖清单或 CVE 编号。
23. 报告全文禁止出现三个连续英文句点；需要省略非关键字段时写“省略非关键字段”。
24. 同根因多入口遵守 `../java-shared/VULNERABILITY_GROUPING.md`。

## 文件命名

```text
{project_name}_xxe_audit_{YYYYMMDD_HHMMSS}.md
```
