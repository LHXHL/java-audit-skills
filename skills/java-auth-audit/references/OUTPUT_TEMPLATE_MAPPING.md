# 路由鉴权映射表模板

只在生成 `{project_name}_auth_mapping_{YYYYMMDD_HHMMSS}.md` 时读取本文件。映射表用于呈现已知入口的鉴权状态和证据来源，不写漏洞详情、Burp Suite 请求或 payload。

## 硬边界

- 固定 4 个二级章节，章节名和顺序不得改变。
- 可以记录 `待验证`、`不可确认` 和 `非漏洞`，但只能以路由映射或统计形式出现。
- 不输出漏洞复现步骤、HTTP 请求、payload、CVE、CVSS、修复版本、输出自检或模型验收。
- 未使用 route mapper 时必须在第 1 节说明覆盖限制，不得声称已覆盖所有路由。
- `条件成立` 不是路由鉴权状态，不得写入映射表的鉴权状态列；路由状态只能使用第 1 节定义的状态。

## 文件结构

```markdown

输出文件: {project_name}_auth_mapping_{YYYYMMDD_HHMMSS}.md

## 1. 鉴权状态说明

| 状态 | 定义 | 输出位置 |
|------|------|----------|
| 公开 | 明确设计为无需登录，且无敏感影响 | 映射表 |
| 受保护 | 有认证和所需授权/对象归属校验 | 映射表 |
| 仅认证 | 只确认登录校验，角色/对象权限未确认或缺失 | 映射表，满足主报告门槛时在备注关联 AUTH 编号 |
| 无鉴权 | 未发现有效认证或授权保护 | 映射表，满足主报告门槛时在备注关联 AUTH 编号 |
| 待验证 | 有候选模式但缺运行时、父类、AOP、角色或对象证据 | 映射表和 README |
| 不可确认 | 关键源码、配置或反编译结果缺失 | 映射表和 README |
| 非漏洞 | 已确认保护有效或入口应公开且无敏感影响 | 映射表 |

覆盖说明:

- route mapper: {已使用/未使用；路径}
- 独立扫描范围: {配置、Controller、Servlet、Action、WebService、JAR/反编译}
- 已知限制: {动态路由、数据库权限、第三方网关、未反编译类等}

| {模块} | `{path 或 endpoint}` | `{GET/POST/SOAP/RPC}` | {公开/受保护/仅认证/无鉴权/待验证/不可确认/非漏洞} | {Shiro/Spring Security/JWT/Session/Filter/Interceptor/自定义/无/未知} | {role/permission/owner/tenant/无/未知} | {文件:行或配置路径} | {简短限制或关联 AUTH 编号} |
```
