# 鉴权审计 README 模板

只在生成 `{project_name}_auth_README_{YYYYMMDD_HHMMSS}.md` 时读取本文件。README 说明审计范围、方法、局限、待验证项和后续建议，不写可复制攻击请求。

## 硬边界

- 固定 6 个二级章节，章节名和顺序不得改变。
- README 中不要使用三级标题。
- 不输出具体漏洞详情、Burp Suite 请求、payload、CVE、CVSS、修复版本、输出自检、模型验收或 Claude 运行状态。
- 待验证项、不可确认项、加固建议可以放在 README，但不能写成已确认漏洞。
