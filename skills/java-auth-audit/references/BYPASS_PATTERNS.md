# 鉴权绕过模式参考

只在已识别鉴权链路、需要验证路径/参数/方法/编码绕过时读取本文件。这里提供候选 payload 类别和判断条件；漏洞结论必须结合数据流和后续拦截层。

## 路径绕过类别

| 类别 | 示例 | 判断条件 |
|------|------|----------|
| 分号参数 | `/admin;.js`, `/public;/../admin` | 鉴权使用原始 URI，路由层会去除或规范化分号部分 |
| 路径穿越 | `/public/../admin`, `/admin/%2e%2e/` | 白名单先匹配，后续路由解析到敏感入口 |
| 双斜杠/点路径 | `//admin`, `/./admin` | 鉴权层和路由层规范化不一致 |
| 编码差异 | `%2f`, `%252f`, `%5c` | 解码次数或斜杠处理不一致 |
| 大小写 | `/ADMIN` vs `/admin` | 文件系统/容器/应用大小写处理不一致 |
| 后缀 | `/admin.action`, `/admin;.css` | 后缀白名单和真实路由不一致 |

## 参数/方法绕过类别

| 类别 | 示例 | 判断条件 |
|------|------|----------|
| HTTP 方法切换 | GET 改 POST/OPTIONS | 鉴权只拦截部分方法，路由仍接受 |
| 方法覆盖 | `_method=DELETE`, `X-HTTP-Method-Override` | 框架启用 method override |
| 参数覆盖 | `role=user&role=admin` | 鉴权和业务取值策略不同 |
| 隐藏参数 | `isAdmin=true`, `tenantId=` | 参数影响权限或数据范围 |

## 必须验证的问题

- payload 是否到达同一个业务入口。
- 绕过的是登录、角色、权限还是静态资源白名单。
- 绕过后是否还有 Filter/Interceptor/方法注解/业务归属校验。
- 成功响应是否包含敏感数据或执行敏感动作。

## 结果标注

- `已确认`: 有代码路径证据，且无后续有效拦截；或有可复现响应。
- `待验证`: 代码存在候选模式，但运行时解析差异未确认。
- `不可利用`: 后续层有效拦截或 payload 不到达敏感入口。
- `环境依赖`: 依赖容器、代理、框架版本或部署配置。

## Gotchas

- payload 列表不是漏洞列表。
- 白名单绕过常只影响已登录用户的细粒度权限，不一定影响未登录访问。
- 代理/Nginx/Tomcat/Spring 的路径规范化链路必须一起看。
