# 鉴权绕过模式参考

只在已识别鉴权链路、需要验证路径、参数、方法、编码或 matcher 绕过时读取本文件。这里提供候选 payload 类别和判断条件；漏洞结论必须结合数据流和后续拦截层。

## 路径绕过类别

| 类别 | 示例 | 判断条件 |
|------|------|----------|
| 分号参数 | `/admin;.js`, `/public;/../admin` | 鉴权层和路由层对分号处理不同 |
| 路径穿越 | `/public/../admin`, `/admin/%2e%2e/` | 白名单先匹配，最终路由到敏感入口 |
| 双斜杠/点路径 | `//admin`, `/./admin` | 规范化前后匹配结果不同 |
| 编码差异 | `%2f`, `%252f`, `%5c` | 解码次数或斜杠处理不同 |
| 大小写 | `/ADMIN` 与 `/admin` | 容器、文件系统或框架大小写处理不同 |
| 后缀 | `/admin.action`, `/admin;.css` | 后缀白名单和真实路由不一致 |

## 参数和方法绕过类别

| 类别 | 示例 | 判断条件 |
|------|------|----------|
| HTTP 方法切换 | GET 改 POST/OPTIONS | 鉴权只拦截部分方法，路由仍接受 |
| 方法覆盖 | `_method=DELETE`, `X-HTTP-Method-Override` | 框架启用 method override |
| 参数覆盖 | `role=user&role=admin` | 鉴权和业务取值策略不同 |
| 隐藏参数 | `isAdmin=true`, `tenantId={id}` | 参数影响权限或数据范围 |

## 升格条件

进入主报告前必须确认：

- payload 到达同一个敏感业务入口。
- 绕过目标是登录、角色、权限、对象归属或白名单。
- 后续 Filter、Interceptor、注解、业务校验不会补救。
- 成功响应会产生敏感数据读取、敏感动作执行或权限边界突破。

## 状态标注

- `确认漏洞`：代码路径证据或授权测试证据完整，且无后续有效拦截。
- `条件成立`：代码证据完整，只剩明确外部条件。
- `待验证`：候选模式存在，但运行时解析、部署路径或后续拦截未知。
- `不可确认`：关键源码、配置或反编译结果缺失。
- `非漏洞`：payload 不到达敏感入口，或后续层有效拦截。

## Gotchas

- payload 列表不是漏洞列表。
- 白名单绕过常只影响已登录用户的细粒度权限，不一定是未登录访问。
- 代理、容器、框架、应用四层路径规范化要一起看。
- 待验证 payload 不写进主报告的 Burp Suite 请求。
