# 鉴权门禁参考

本文件不列具体过滤器、拦截器、网关或框架 API。统一识别 `AUTH_GATE`。

## 必查证据

- 注册来源和覆盖范围。
- 匹配路径或 operation 的规则。
- 放行规则、静态资源、健康检查、回调或内部接口例外。
- 身份认证和权限授权是否分离。
- 放行后是否还有业务层 `POLICY_RULE`。

## 常见缺口

- 只覆盖一类入口，未覆盖服务根、dispatcher 或动态 operation。
- 路径解析与 route mapper 不一致。
- 放行规则过宽。
- 只认证身份，未校验资源归属。

缺覆盖时写待验证或确认漏洞，取决于入口可达性和业务影响证据。
