# 鉴权信息梳理报告模板

用户要求“梳理当前源码下鉴权信息”“整理权限配置”“输出路由鉴权映射”时使用本模板。报告只写从源码、配置或反编译产物能证明的认证授权机制；不能把框架默认行为当项目事实。

```markdown
# 【项目名称】Java 鉴权信息梳理报告

生成时间: 【YYYY-MM-DD HH:MM:SS】
梳理目标: 【路径或目标说明】
输入类型: 【源码/JAR/WAR/class/反编译产物/diff/混合】
审计工作目录: 【java-audit-workspace 路径】

## 1. 梳理概述

| 项目 | 内容 |
|---|---|
| 梳理范围 | 【范围】 |
| 认证机制 | 【Session/JWT/Shiro/Spring Security/自定义/未知】 |
| 授权机制 | 【注解/配置/拦截器/过滤器/代码判断/未知】 |
| 使用材料 | 【源码/配置/反编译结果/diff】 |
| 覆盖结论 | 【覆盖情况摘要】 |

## 2. 鉴权机制

| 编号 | 类型 | 实现位置 | 关键规则 | 证据位置 | 备注 |
|---|---|---|---|---|---|
| AUTH-001 | 【认证/授权/放行/异常处理】 | 【类/配置/方法】 | 【规则】 | 【文件:行号 或 反编译文件+方法】 | 【备注】 |

## 3. 路由/入口鉴权映射

| 编号 | 路由/入口 | HTTP 方法 | Handler | 鉴权状态 | 权限要求 | 证据位置 | 备注 |
|---|---|---|---|---|---|---|---|
| MAP-001 | 【路径】 | 【GET/POST/ALL/N/A】 | 【类.方法】 | 【已鉴权/放行/未知/条件鉴权】 | 【角色/权限/表达式/无/未知】 | 【文件:行号】 | 【备注】 |

## 4. 风险观察与待确认项

| 编号 | 观察 | 当前证据 | 缺失证据 | 下一步 |
|---|---|---|---|---|
| AUTH-LEAD-001 | 【观察】 | 【已有证据】 | 【缺失证据】 | 【下一步确认】 |

## 5. 覆盖范围与限制

- 已覆盖: 【模块、包、配置、反编译范围】
- 未覆盖: 【未覆盖原因】
- 不确定项: 【动态权限、外部网关、配置中心、运行时开关、反编译缺失等】
- 后续建议: 【需要补充的运行配置或源码材料】
```

## 填写规则

- 不得保留 `【...】` 占位符。
- 不得把“未发现鉴权代码”直接写成“未授权漏洞”；这里只输出鉴权梳理，不确认漏洞。
- 鉴权状态不明确时写 `未知` 或 `条件鉴权`，并说明缺失证据。
- 放行规则必须有配置、注解、代码分支或过滤器链证据。
