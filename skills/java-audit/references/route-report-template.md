# 路由信息梳理报告模板

用户要求“梳理当前源码的路由信息”“整理接口入口”“输出路由报告”时使用本模板。报告只写从源码、配置或反编译产物中能证明的入口；不能凭命名、猜测或框架习惯补路由。

```markdown
# 【项目名称】Java 路由信息梳理报告

生成时间: 【YYYY-MM-DD HH:MM:SS】
梳理目标: 【路径或目标说明】
输入类型: 【源码/JAR/WAR/class/反编译产物/diff/混合】
审计工作目录: 【java-audit-workspace 路径】

## 1. 梳理概述

| 项目 | 内容 |
|---|---|
| 梳理范围 | 【范围】 |
| 识别框架 | 【Spring MVC/Servlet/Struts/JAX-RS/WebService/其他】 |
| 使用材料 | 【源码/配置/反编译结果/diff】 |
| 路由来源 | 【注解/web.xml/注册代码/框架配置/其他】 |
| 覆盖结论 | 【覆盖情况摘要】 |

## 2. 路由汇总

| 类型 | 数量 | 说明 |
|---|---:|---|
| Controller/Handler 路由 | 【数量】 | 【说明】 |
| Servlet/Filter 入口 | 【数量】 | 【说明】 |
| WebService/RPC 入口 | 【数量】 | 【说明】 |
| 未确认入口 | 【数量】 | 【说明】 |

## 3. 路由详情

| 编号 | HTTP 方法 | 路径/入口 | Handler | 参数 | 鉴权线索 | 证据位置 | 备注 |
|---|---|---|---|---|---|---|---|
| ROUTE-001 | 【GET/POST/ALL/N/A】 | 【路径】 | 【类.方法】 | 【参数】 | 【注解/过滤器/未知】 | 【文件:行号 或 反编译文件+方法】 | 【备注】 |

## 4. 覆盖范围与限制

- 已覆盖: 【模块、包、配置、反编译范围】
- 未覆盖: 【未覆盖原因】
- 不确定项: 【动态注册、网关转发、条件配置、反编译缺失等】
- 后续建议: 【需要补充的运行配置或源码材料】
```

## 填写规则

- 不得保留 `【...】` 占位符。
- 路由数量必须来自实际条目统计；不能写“大约”“若干”。
- 无法确认 HTTP 方法时写 `N/A` 或 `ALL`，并在备注说明原因。
- 动态路由、条件注册或反编译不完整时，放入“未确认入口”或限制说明。
