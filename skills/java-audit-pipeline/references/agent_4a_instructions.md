# Agent-4a：路由优先级分级

## 职责

`agent-4a` 只根据路由证据和鉴权证据生成追踪优先级。它不确认漏洞，不解释组件 CVE，也不筛掉 P0/P1 路由。

## 输入

- `{output_path}/route_mapper/README.md` 和各模块详情文件。
- `{output_path}/auth_audit/` 中的鉴权映射、无鉴权路由、鉴权绕过发现。
- 可选：`{output_path}/cross_analysis/component_version_evidence.md`，仅用于标注“组件触发面待核查”，不得改写鉴权状态。

## 分级规则

| 优先级 | 条件 | 下游动作 |
|---|---|---|
| P0 | `java-auth-audit` 明确判定无鉴权的路由 | 全量进入调用链追踪 |
| P1 | `java-auth-audit` 明确给出鉴权绕过发现或条件成立发现的路由 | 全量进入调用链追踪，并透传上游验证材料索引 |
| P2 | 有鉴权或鉴权状态未形成高危结论的路由 | P0+P1 为空时用于兜底追踪，或由用户指定抽样 |
| C1 | 组件版本证据提示可能存在请求触发面，但尚未被专项 skill 验证 | 可作为附加追踪候选，不得标记为鉴权绕过 |

## 强制要求

- `待追踪路由列表` 必须包含全部 P0 和 P1；不得自行裁剪。
- C1 不计入 P1，不写“可绕过鉴权”。
- 追踪理由不写具体 payload，不复述利用步骤。
- 数量必须来自实际表格或文件计数；未知写“未统计”，不要写约数。
- 不得写尾随加号、范围数量、波浪线数量或估算时长；无法精确枚举时写“未精确统计”，不要把未枚举操作数写成确定覆盖。
- 没有 P0/P1 时必须保留 P2 兜底说明，供 `agent-5` 判断是否追踪。
