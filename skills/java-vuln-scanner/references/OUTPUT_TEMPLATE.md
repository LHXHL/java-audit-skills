# 组件版本命中报告模板

硬约束：

1. 报告必须且只能包含 6 个编号章节。
2. 状态枚举只允许：`版本命中`、`触发面待核查`、`环境条件待确认`、`不可确认`、`未命中`。
3. 不得输出业务风险成立、验证成功、通用评分、具体版本建议、攻击字符串或可复制验证材料。
4. 规则命中必须来自扫描脚本 JSON 返回的 `RULE_ID`。
5. 统计数字必须精确；无法确认写 `不可确认`。
6. 来源文件不得使用省略写法。
