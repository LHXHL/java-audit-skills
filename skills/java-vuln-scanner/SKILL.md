# Java Vulnerability Scanner

## 当前定位

`java-vuln-scanner` 是组件版本证据层。它从构建文件、依赖清单、部署包和归档元数据中提取组件与版本，使用本地规则数据匹配 `RULE_ID`，并输出触发面待核查建议。

本 skill 只证明“组件版本命中或不可确认”，不证明业务漏洞真实成立，不输出攻击字符串、复核 payload、通用评分或具体升级版本。

## 触发条件

- 用户要求扫描依赖、组件版本、CVE/规则命中或 SCA 风险。
- 用户提供构建文件、依赖清单、归档包或部署产物。
- pipeline 需要组件版本证据和专项交接建议。

不触发：

- 用户要求证明某个接口漏洞真实成立。
- 用户要求生成验证请求、攻击字符串或 payload。
- 用户要求审计业务 sink、鉴权或调用链。
- 用户要求最新实时公告或具体升级版本；本地规则不是实时情报源。

## 工作流

1. 确定扫描范围和依赖来源。
2. 运行 `scripts/scan_dependencies.py` 获取 JSON 结果。
3. 只引用脚本 JSON 中实际返回的组件、版本和规则命中。
4. 按 `COMPONENT_ID + COMPONENT_VERSION + RULE_ID` 聚合。
5. 根据 `references/TRIGGER_SURFACE_GUIDE.md` 写触发面待核查和 `DISPATCH_TO`。
6. 使用 `references/OUTPUT_TEMPLATE.md` 输出 6 个编号章节。

## 成功标准

- 依赖来源可追溯到具体文件、模块和版本。
- 统计数字来自脚本或可复核清点。
- 命中状态只使用：版本命中、触发面待核查、环境条件待确认、不可确认、未命中。
- 清楚说明版本命中不等于业务风险成立。
- 不输出可复制验证材料、攻击链、具体升级版本、内部规则编号或 validator 结果。

## Hard Rules

1. 没有依赖来源和版本，不得输出规则命中。
2. 脚本 JSON 未返回的规则不得写入正式报告。
3. 本地规则不是实时情报，不得声称最新或完整覆盖。
4. 命中项需要业务验证时，只写交接方向，不在本 skill 内补漏洞链。
5. 最终回复只给报告路径和固定结论，不复述命中清单。

## 按需读取

- 规则使用边界：`references/RULE_USAGE.md`
- 触发面分流：`references/TRIGGER_SURFACE_GUIDE.md`
- 输出模板：`references/OUTPUT_TEMPLATE.md`

| 类型 | 场景 | 预期 |
|------|------|------|
| 正例 | 扫描部署包组件版本命中 | 输出版本证据和触发面待核查 |
| 正例 | pipeline 需要组件风险输入 | 输出专项交接建议 |
| 反例 | 要求验证某接口漏洞 | 切换到对应专项 |
| 反例 | 要求最新升级版本 | 说明本地规则局限 |
