# 路径遍历判定

## 风险成立条件

1. `REQUEST_PARAM` 影响路径、资源名或间接文件标识。
2. 该值进入 `PATH_JOIN` 或等价路径解析。
3. `NORMALIZATION_GUARD` 缺失、不足或发生在错误位置。
4. 读取目标可被输出或产生敏感后续影响。

## 有效防护

- 固定根目录且规范化后确认仍在根目录内。
- 闭合集合映射到服务端文件 ID。
- 文件名严格枚举。
- 后缀、类型、大小和权限检查与根目录约束同时生效。

## 不充分防护

- 只替换个别字符。
- 只检查非空、长度或后缀。
- 规范化后未比较根目录。
- 检查发生在路径再次拼接之前。

## 结论状态

缺路径来源、读取实现、防护位置或输出条件时，不得写确认漏洞。
