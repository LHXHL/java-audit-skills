# Burp Suite 与 Payload 验证材料

只在某条文件读取风险已经达到 `确认漏洞` 或 `条件成立` 门槛时读取本文件。

## 生成条件

必须同时满足：

- 真实入口、HTTP 方法、参数名、Content-Type 或 RPC/SOAP operation 已确认。
- 外部输入能影响读取路径、文件名、资源 key 或下载 ID。
- 已定位真实文件读取 sink。
- 防护缺失或不足已经由代码证据证明。
- payload 只验证当前风险，不做批量读取或破坏性操作。

以下情况不得生成 Burp Suite 请求或 payload：

- 状态为 `待验证`、`不可确认` 或 `非漏洞`。
- 只有参数名疑似文件路径，但未追到读取 sink。
- 只有 sink，缺入口和可控参数。
- 只有 class 名，缺反编译方法体。
- 只发现静态资源、固定帮助文档、固定 classpath 资源。

## 请求写法

- 使用 Burp Repeater 可直接粘贴的 HTTP 请求块。
- Host、Cookie、Token、CSRF、对象 ID、文件 ID 使用占位符。
- 保留真实路径、参数名、Content-Type 和必要 header。
- 不写生产真实路径、真实敏感文件内容、批量枚举脚本。
- 路径值、文件名值和目标文件值必须写成 `{{...}}` 占位符；不要写真实系统敏感路径或真实应用配置文件路径。

## Payload 写法

优先使用低风险测试目标：

- 应用内安全测试文件：`{{safe_test_file}}`。
- 测试环境预置文件：`{{known_test_file}}`。
- 允许开发单位自行替换的系统公共文件占位符：`{{linux_safe_probe}}`、`{{windows_safe_probe}}`。

payload 表至少包含：

| 目的 | 变体 | 预期观察 |
|------|------|----------|
| 基线读取 | `{{allowed_file}}` | 返回正常文件或下载响应 |
| 目录穿越 | `{{traversal_payload}}` | 若返回受控测试文件内容差异、响应长度差异或服务端日志显示越界读取，则支持结论 |
| 防护否定 | `{{blocked_payload}}` | 若返回 400/403/错误码或白名单拒绝，则可能否定风险 |

## 安全限制

- 只在授权测试环境使用。
- 不读取真实敏感文件内容。
- 不批量遍历路径。
- 不使用自动化利用脚本。
- 写清什么响应支持结论，什么响应否定结论。
