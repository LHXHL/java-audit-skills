# 组件与链条条件

本文件定义对象解码链条的完整性条件。

## 完整链条要求

| 条件 | 说明 |
|------|------|
| `ENTRY_ROUTE` | 外部可达入口真实存在 |
| `OBJECT_DECODER` | 真实对象解码 sink 可达 |
| `TYPE_METADATA` | 攻击者可影响类型、对象图或多态字段 |
| `AUTO_TRIGGER_METHOD` | 解码后会自动触发可控对象行为 |
| `DANGEROUS_PRIMITIVE` | 链条到达命令、反射、文件、网络、模板、脚本或表达式能力 |
| `TYPE_FILTER` | 未被白名单、过滤器、签名、加密、固定类型或环境限制阻断 |

## 结论

- 全部条件具备：确认漏洞或条件成立。
- 缺入口、缺 decoder、缺类型控制、缺自动触发或缺危险原语：待验证/不可确认/非漏洞。
- 组件存在但链条未闭合：只写候选，不写漏洞。

## 输出要求

报告必须把缺失条件写清楚，不能用“存在组件”替代链条证明。
